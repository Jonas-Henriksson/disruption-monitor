# Executive Summary — Design Spec

**Date:** 2026-04-17
**Status:** Approved

## Problem

The app has all the data an executive needs (weekly summary, severity breakdown, evolution tracking, BU exposure) but it's scattered across a sidebar, a modal, and per-event cards. A VP opening the app at 8am Monday has to click around to answer "should I worry?" The WeeklyBriefing modal is never opened because nobody clicks modals on purpose.

## Solution

Two-part feature:

1. **In-app: Collapsible Executive Hero Panel** above the feed
2. **Push: Monday 8am Teams digest** with the same content, deep-linked to the app

## Part 1: Executive Hero Panel

### Placement

Sits between DeltaBanner and FeedList. Replaces the current DeltaBanner entirely (the hero panel subsumes its function).

```
TopBar
[Scan Progress Bar]
[Executive Hero Panel]        ← NEW (replaces DeltaBanner)
  ├── Risk Posture Bar        (10-second answer)
  ├── Three-Column Layout     (30-second scan)
  │   ├── Actively Bleeding
  │   ├── Escalating
  │   └── Recently Resolved
  └── Collapse chevron
[FeedList (65%) | Sidebar (35%)]
```

### Auto-expand logic

- **Expanded by default** when: first visit of the day, or first visit in 8+ hours
- **Collapsed by default** when: user already viewed it this session
- Collapse state stored in `sessionStorage` (resets on new tab/day)
- User can always toggle via chevron

### Content (when expanded, ~160px tall)

#### Row 1: Risk Posture Bar (10-second answer)

Single horizontal bar spanning full width:

```
[Risk Level: ELEVATED]  3 Critical  |  12 High  |  8 Medium  |  2 Low    [Mon 14 Apr — Mon 21 Apr]
```

- Risk level derived from Critical count: 0 = STABLE (green), 1-2 = ELEVATED (amber), 3+ = HIGH (red)
- Severity counts with colored dots, inline
- Date range (last 7 days) right-aligned
- AI one-liner underneath: "Port congestion in SE Asia converging with EU steel tariffs — 3 manufacturing sites on shortened buffer stock." Generated by Claude Sonnet, max 30 words, pre-computed during scan.

#### Row 2: Three-Column Layout (30-second scan)

| Actively Bleeding | Escalating | Recently Resolved |
|---|---|---|
| Top 3 Critical/High events | Top 3 events trending up | Top 3 archived this week |
| Each: title (truncated), region, affected site count | Each: title, severity delta arrow, days active | Each: title, was-severity, resolution note |

- Each column capped at 3 items. No scrolling.
- Each item is clickable — scrolls feed to that event and expands it.
- "Actively Bleeding" sorted by severity score descending.
- "Escalating" = events where latest severity > first severity in the period.
- "Recently Resolved" = events archived in last 7 days, most recent first.

#### Row 3: BU Impact Strip (optional, shows if BU data available)

Compact horizontal strip showing per-BU exposure:

```
Industrial: 5 events (2 Crit)  |  Aerospace: 3 events (1 Crit)  |  Seals: 1 event  |  Lubrication: 0
```

- Only BUs with active events shown
- Clicking a BU filters the feed

### Collapsed state (1 line, ~32px)

When collapsed, shows a single-line summary:

```
▸ Risk: ELEVATED — 3 Critical, 12 High, 8 Medium    [Expand summary]
```

### Data sources

All data already exists in the backend:

| Content | Source |
|---|---|
| Severity counts | `GET /scans/latest/{mode}` → count by severity |
| Risk level | Derived from Critical count (frontend logic) |
| Top events | `GET /events?status=active&limit=50` sorted by severity |
| Escalating events | `GET /events/weekly-summary` → `escalated_events` |
| Resolved events | `GET /events/weekly-summary` → `resolved_events` |
| BU exposure | `GET /exposure/bu-summary` |
| AI one-liner | New: pre-computed during scan, stored alongside weekly summary |

### New backend work

**One new field:** `executive_one_liner` — a single sentence (max 30 words) summarizing the overall risk posture. Generated by Sonnet during the scan pipeline, stored in the `scan_records` table or a new `executive_summaries` table.

**Prompt:**
```
You are a supply chain risk analyst. Given the current active disruptions below,
write exactly ONE sentence (max 30 words) answering: "What should a VP know right now?"
Focus on convergent risks and operational impact. No hedging.

Active Critical events: {critical_events}
Active High events: {high_events}
Escalating events: {escalating_events}
```

**Generation cadence:** Once per disruptions scan (every 15 min). Cached. Frontend reads latest.

## Part 2: Monday Teams Digest

### Trigger

Scheduler fires at 08:00 UTC every Monday (configurable via `TARS_DIGEST_TIME` and `TARS_DIGEST_DAY`).

### Content

Same structure as the hero panel but formatted as a Teams Adaptive Card:

1. **Header:** "SC Hub Weekly Risk Digest — Week of {date}"
2. **Risk posture:** Level + severity counts + AI one-liner
3. **Actively Bleeding:** Top 3 Critical/High with deep links
4. **Escalating:** Top 3 with severity arrows
5. **Action Required:** Overdue tickets with owner tags
6. **Link:** "Open SC Hub" button → CloudFront URL

### Delivery

Uses existing `teams_channel.py` Incoming Webhook infrastructure. New function: `send_weekly_digest()`. Called by a new scheduler loop `_digest_loop()` that runs once per week.

### Telegram fallback

Also send via Telegram if configured, using the existing `send_telegram_message()` function. Same content, formatted as plain text with markdown.

## What we're NOT building

- No separate route or page
- No "executive mode" toggle
- No PDF export (yet)
- No email delivery (Teams + Telegram cover the push use case)
- No per-BU AI narrative (the one-liner is enough for now)
- No daily digest push (weekly only; the in-app panel is always fresh)

## Component structure

### Frontend

```
frontend/src/v3/components/
  ExecutiveHero.tsx          ← NEW (~250 lines)
    ├── RiskPostureBar       (inline, not separate component)
    ├── ThreeColumnGrid      (inline)
    └── BUStrip              (inline)
```

Single component. No over-abstraction. Receives `items`, `weeklySummary`, `buExposure` as props from V3App.

### Backend

```
backend/app/services/
  executive.py               ← NEW (~80 lines)
    └── generate_executive_one_liner(events) -> str

backend/app/services/scheduler.py
  └── _digest_loop()         ← NEW (sends Monday Teams digest)

backend/app/routers/events.py
  └── GET /events/executive-summary   ← NEW (returns hero panel data)
```

### New API endpoint

`GET /api/v1/events/executive-summary`

Returns:
```json
{
  "risk_level": "ELEVATED",
  "one_liner": "Port congestion converging with EU steel tariffs...",
  "severity_counts": { "Critical": 3, "High": 12, "Medium": 8, "Low": 2 },
  "actively_bleeding": [ /* top 3 events */ ],
  "escalating": [ /* top 3 events */ ],
  "recently_resolved": [ /* top 3 events */ ],
  "bu_exposure": [ /* per-BU summary */ ],
  "period": { "from": "2026-04-10", "to": "2026-04-17" },
  "generated_at": "2026-04-17T08:00:00Z"
}
```
